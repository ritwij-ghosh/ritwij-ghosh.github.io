AI Investment Banking Research Agent

Overview

This project is a LangChain/LangGraph agentic system designed to automate the research workflows common in investment banking and equity analysis. Given a portfolio of holdings, the agent continuously monitors news and price data, identifies material events, ranks their relevance to the portfolio, and surfaces alerts with LLM-generated research briefs.

Multi-Source Signal Aggregation

The agent pulls from two primary data sources: yfinance for real-time and historical price/fundamental data, and NewsAPI for news articles across financial publications. For each holding, the agent retrieves recent articles, filters for material events (earnings surprises, M&A activity, regulatory filings, analyst rating changes), and scores their estimated impact on the user's position size and portfolio weighting.

LangGraph's stateful graph execution model is used to orchestrate the multi-step research loop: fetch → filter → score → summarize → write → alert. Each node in the graph is a discrete function with typed inputs/outputs, making the pipeline inspectable and extensible.

Knowledge Graph (Neo4j)

Research findings are written to a continuously updated knowledge graph built on Neo4j (with NetworkX used for in-memory graph operations during processing). Nodes represent tickers, companies, industries, analysts, and news events. Edges encode relationships: "covers," "competes with," "reported by," "affects sector," and so on.

This graph structure enables cross-portfolio queries that flat databases cannot efficiently support — for example, identifying second-order impacts (a supplier disruption at one holding affecting another holding's supply chain) or clustering correlated risk events by sector.

Research Brief Generation

For each material alert, the agent generates a structured research brief using LLM summarization over the retrieved articles and knowledge graph context. Briefs follow a consistent template: event summary, affected holdings, estimated impact direction, supporting evidence, and recommended follow-up actions. These are delivered as push notifications and stored for historical review.

Key Features

Neo4j knowledge graph with relationship traversal
Multi-source signal aggregation (yfinance + NewsAPI)
LangGraph stateful research loop
LLM-generated research briefs
